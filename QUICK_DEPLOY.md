# üéØ Quick Deployment Reference

## ‚ö° 3-Minute Quick Start

### Step 1: Run Setup (1 minute)
```bash
# Windows
setup-production.bat

# Linux/Mac
chmod +x setup-production.sh && ./setup-production.sh
```
This auto-generates secure JWT secrets and creates your `.env` file.

---

### Step 2: Configure Environment (1 minute)
```bash
# Edit .env file
notepad .env     # Windows
nano .env        # Linux/Mac
```

**Must replace these values:**
- `DATABASE_URL` - Your production database
- `GITHUB_API_KEY` - From https://github.com/settings/tokens
- `AWS_ACCESS_KEY_ID` & `AWS_SECRET_ACCESS_KEY` - For S3 storage
- `ALLOWED_ORIGINS` - Your domain(s)

**Optional (can skip for quick test):**
- `ETH_RPC_URL`, `SMART_CONTRACT_ADDRESS`, `PRIVATE_KEY` - Blockchain features
- `LINKEDIN_API_KEY` - LinkedIn verification

---

### Step 3: Deploy (1 minute)
```bash
docker-compose -f docker-compose.production.yml up -d --build
```

**That's it!** Access at http://localhost:3000

## üÜò Common Issues

**Backend won't start:**
```bash
# Check DATABASE_URL is set
cat .env | grep DATABASE_URL

# Use included PostgreSQL instead
docker-compose -f docker-compose.production.yml up postgres -d
```

**Port already in use:**
```bash
# Change ports in docker-compose.production.yml
# 8000 ‚Üí 8001, 3000 ‚Üí 3001, etc.
```

**Can't connect to database:**
```bash
# Use included PostgreSQL container
DATABASE_URL=postgresql://postgres:postgres@postgres:5432/resume_verify
```
